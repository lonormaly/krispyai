# `TenantConfig`

everything a tenant needs, keyed by `tenantId` (default `"self"`) in KV under `tenant:<id>`.
for `"self"`, `botToken`/`chatId` come from Worker **secrets** and the rest from KV; any
other tenant reads the whole thing from KV.

```ts
interface TenantConfig {
  botToken: string; // Telegram bot token (BotFather)
  chatId: string; // target supergroup id WITH topics, e.g. -1001234567890
  systemPrompt?: string; // optional system-prompt override
  model?: string; // optional model override
  onboardingStep?: number; // onboarding progress (optional; mirrors cloud)
  onboardingComplete?: boolean;
  forms?: FormSpec[]; // lead forms
  connectors?: Connector[]; // delivery / CTA connectors
  theme?: WidgetTheme; // widget appearance
  operators?: Operator[]; // humans to @mention on handoff (auto-learned; NEVER public)
}
```

`getTenant()` only treats a config as usable once **both** `botToken` and `chatId` are
present; a partial config (e.g. a prompt saved before creds) still round-trips through KV.

## `FormSpec`

```ts
interface FormSpec {
  id: string; // referenced by [!FORM:<id>]
  title: string; // card heading, e.g. "book a call"
  fields: FormField[];
  connectorIds?: string[]; // which connectors receive this lead; default = ALL configured
  successText?: string; // shown in the collapsed card after submit; widget has a default
}

interface FormField {
  name: string; // machine key, e.g. "budget"
  label: string; // visitor-facing
  type: "text" | "email" | "tel" | "textarea" | "select";
  required?: boolean;
  options?: string[]; // select only
}
```

## `Connector`

```ts
type ConnectorType = "email" | "telegram" | "whatsapp" | "instagram";

interface Connector {
  id: string;
  type: ConnectorType;
  toAddress?: string; // email
  phone?: string; // whatsapp — E.164 digits, no "+"
  profileUrl?: string; // instagram
  // telegram uses the top-level botToken/chatId — no per-connector creds
}
```

`telegram` + `email` are delivered server-side; `whatsapp` + `instagram` are CTA links only.
see [lead forms + connectors](/docs/guides/lead-forms-and-connectors).

## `Operator`

```ts
interface Operator {
  id: number; // Telegram user id — enough for a text_mention (no public @username needed)
  name?: string; // display name (first_name), used as the mention label when no username
  username?: string; // public @username (preferred — a plain @mention, no entity)
}
```

the humans the bot `@mentions` on **handoff** (the "quiet ops" workflow — mute the group,
get pinged only when a person is needed). **Auto-learned:** whoever replies (non-bot) in a
managed topic is upserted into this list, capped at 10 (FIFO). Zero-config — no need to set
it by hand, though you can via `POST /api/tenant/config`. With an empty list the handoff
alert still fires, just without a mention.

`operators` is **secret-adjacent** (Telegram user ids) and is **never** exposed to the public
`GET /api/widget/config`. see [connect Telegram](/docs/guides/connect-telegram) and
[security](/docs/security).

## `WidgetTheme`

```ts
interface WidgetTheme {
  primaryColor?: string; // header + visitor bubble + send button. Default gold #e39a2b
  launcherColor?: string; // launcher only (defaults to primaryColor)
  glowColor?: string; // hex → launcher glow/sparkle/pulse tint. Unset = no glow layer at all
  position?: "br" | "bl"; // bottom-right | bottom-left. Default "br"
  avatar?: string; // "buttr" (inline default) | https URL | data:image/… URI (≤48KB)
  greeting?: string; // first bot bubble on open
  headerTitle?: string; // header text (supersedes data-title)
  tagline?: string; // header sub-line ("usually replies in minutes")
  radius?: number; // corner radius px, clamped 0–20. Default 14
  font?: string; // CSS font-family stack
  sound?: boolean; // notification ding. Default true
  sparkle?: boolean; // idle sparkle loop on the launcher. Default false
  direction?: "ltr" | "rtl"; // flips bubble corners + input dir. Default "ltr"
  popupText?: string; // proactive teaser-popup copy. Unset = popup off
  timing?: WidgetTiming;
}

interface WidgetTiming {
  // all ms; the widget clamps 0–300000. Each takes effect only when its feature is on.
  launcherDelayMs?: number; // default 0 = no entrance delay
  sparkleAfterMs?: number; // default 10000; inert unless sparkle
  popupDelayMs?: number; // default 8000; inert unless popupText
  popupCooldownHrs?: number; // default 24; inert unless popupText
  autoOpenMs?: number; // default 0 = never — auto-open after inbound msg while closed
}
```

every new knob defaults to unset/off — a tenant that configures nothing gets today's
neutral widget, unchanged.

only these fields are exposed to the public `GET /api/widget/config`; the projection is a
literal whitelist (never a spread of the config), so `botToken`/`chatId`/`systemPrompt` are
structurally excluded. see [security](/docs/security).

## write caps

`POST /api/tenant/config` hard-rejects out-of-bounds configs at the trust boundary
(nothing invalid ever reaches KV):

| rule                                                                                 | response                                 |
| ------------------------------------------------------------------------------------ | ---------------------------------------- |
| `theme.avatar` over 48KB                                                             | `413 { error: "avatar_too_large" }`      |
| `theme.avatar` not `"buttr"` / `https://…` / `data:image/(png\|webp\|jpeg);base64,…` | `400 { error: "avatar_scheme_invalid" }` |
| a connector CTA url (`profileUrl`/`url`) not `https://`                              | `400 { error: "cta_url_not_https" }`     |
| `kbSources` total text over 100K chars                                               | `413 { error: "kb_sources_too_large" }`  |

## KV keys

| key                              | value                                                          |
| -------------------------------- | -------------------------------------------------------------- |
| `tenant:<id>`                    | the `TenantConfig` JSON blob                                   |
| `session:<tenant>:<sessionId>`   | the Telegram thread id for that session                        |
| `thread:<tenant>:<threadId>`     | the session id for that thread                                 |
| `usage:<tenant>:<yyyymm>:<kind>` | monthly usage counter — `kind` ∈ `ai` \| `handoff` \| `tokens` |
| `entitlement:<tenant>`           | optional billing snapshot (unused in self-host)                |
